# Presentation Bootstrap

[Скачать архив текущей ветки main (.zip)](https://github.com/idoziru/pitch-llm-agent/archive/refs/heads/main.zip)

Простой проект для сборки презентации с помощью LLM/агента: вы даёте текст и визуальные примеры, агент делает готовую HTML-презентацию.

## Для кого этот проект

Для менеджеров, маркетологов, основателей и команд, которым нужно быстро собрать аккуратную презентацию без сложных инструментов.

## Что вы получите на выходе

- **`index.html`** — готовая презентация (слайды 1280×720, формат 16:9);
- **PNG-файлы слайдов** — для отправки в чатах и документах;
- **PDF-файл** — финальная версия для отправки клиенту или команде.

## Быстрый старт (5 шагов)

1. Положите текст презентации в папку `wording/`.
2. Положите примеры желаемого стиля в папку `visual_refs/`.
3. Запустите агента в корне проекта.
4. Попросите агента создать/обновить `index.html`, `style_brief.md`, `image_manifest.json`.
5. Откройте презентацию в браузере через локальный сервер и нажмите экспорт PNG.

## Важно знать до начала

- Размер каждого слайда фиксированный: **1280×720**.
- Проект работает **без Node.js и без сборки**.
- Текст, цифры и факты из `wording/` нельзя менять без вашего явного разрешения.
- Нельзя открывать `index.html` напрямую через `file://` для экспорта картинок — нужен локальный сервер.

## Как работать с агентом

1. Подготовьте материалы:
   - текст презентации в `wording/`;
   - визуальные примеры в `visual_refs/`.
2. Дайте агенту задачу: собрать презентацию по этим материалам.
3. Агент должен:
   - изучить `AGENTS.md`;
   - прочитать релевантные файлы из `skills/`;
   - создать или обновить `style_brief.md`;
   - создать или обновить `image_manifest.json`;
   - создать или обновить `index.html`.
4. Сгенерируйте изображения по `image_manifest.json` и положите их в `imgs/`.
5. Откройте презентацию в браузере, проверьте слайды и экспортируйте PNG.

## Если делаете вручную (без агента)

1. Создайте `index.html` со слайдами 1280×720.
2. Соблюдайте стиль по материалам из `visual_refs/`.
3. Добавьте изображения в `imgs/`.
4. Запустите локальный сервер и откройте страницу в браузере.
5. Экспортируйте PNG, затем соберите PDF.

## Как открыть презентацию локально

Не открывайте файл напрямую двойным кликом. Откройте через локальный сервер.

Если в проекте есть `scripts/serve.py`:

```bash
python3 scripts/serve.py
```

Если этого файла нет, используйте стандартную команду Python:

```bash
python3 -m http.server 8000
```

После запуска откройте в браузере:

```text
http://localhost:8000
```

## Изображения для слайдов

Файл **`image_manifest.json`** — это список нужных изображений:

- что должно быть на картинке;
- какой формат/размер нужен;
- какой текст использовать для генератора изображений;
- статус: уже готово или пока заглушка.

Пока изображение не готово, на слайде будет понятная заглушка с описанием.

## Сборка PDF из PNG

1. Откройте [ilovepdf.com/jpg_to_pdf](https://www.ilovepdf.com/jpg_to_pdf).
2. Загрузите PNG-файлы слайдов.
3. Отсортируйте по имени (кнопка **A↓Z**).
4. Выберите размер: **Fit (Same page size as image)**.
5. Выберите ориентацию: **Landscape**.
6. Нажмите **Convert to PDF**.

## Частые проблемы и решения

- **Проблема:** экспорт PNG не работает.  
  **Решение:** вы, скорее всего, открыли файл через `file://`. Запустите локальный сервер и откройте `http://localhost:8000`.

- **Проблема:** часть текста или блоков уезжает за границы слайда.  
  **Решение:** попросите агента уменьшить объём текста на слайде или перестроить расположение блоков, чтобы всё помещалось на экране полностью.

- **Проблема:** стиль слайдов «не тот».  
  **Решение:** добавьте более точные референсы в `visual_refs/` и попросите обновить `style_brief.md`.

## Структура проекта

```text
project/

├── README.md
├── .gitignore
├── wording/
├── visual_refs/
├── imgs/
└── skills/
```

## Папки

### wording/

Текст презентации. Рекомендуемый формат — `.md`.

Пример:

```markdown
# Слайд 1. Заголовок

Текст слайда.

Комментарий по желаемому виду блока.
```

### visual_refs/

Визуальные примеры, на которые нужно ориентироваться:

- презентации;
- PDF;
- изображения;
- скриншоты;
- бренд-материалы.

### imgs/

Готовые изображения для подстановки в слайды.

### skills/

Технические инструкции для агента:

- `html-structure.md`
- `png-export-safety.md`
- `slide-layout-safety.md`
- `image-manifest-prompts.md`

## Чеклист перед финальным экспортом

1. Все слайды в формате 1280×720.
2. На слайдах ничего не обрезано, текст и блоки полностью видны.
3. Текст, цифры и факты из `wording/` не изменены без вашего согласования.
4. Все готовые изображения лежат в `imgs/`.
5. Экспорт PNG запускается в браузере через локальный сервер.
