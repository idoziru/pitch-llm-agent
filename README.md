# Presentation Bootstrap

[Скачать архив текущей ветки main (.zip)](https://github.com/idoziru/pitch-llm-agent/archive/refs/heads/main.zip)

Бутстрап-проект для создания презентаций через LLM/coding agent. Агент генерирует HTML-презентацию по готовому тексту и визуальным референсам.

## Что это

Шаблон для сборки бизнес-презентации из готового текста и визуальных референсов.

Визуальный стиль берётся из `visual_refs/`.

Агент создаёт три файла:

- **`style_brief.md`** — описание визуального стиля: типографика, палитра, визуальный язык, правила для иллюстраций. Составляется на основе `visual_refs/`.
- **`image_manifest.json`** — реестр всех изображений презентации: описания, промпты для генерации, статусы (найдено / placeholder).
- **`index.html`** — сама презентация. Один HTML-файл со слайдами 1280×720px, CSS и JS. Работает без сборки и зависимостей.

## Почему HTML, а не изображения

Презентация — это HTML-код и текст, а не картинки. Это даёт несколько преимуществ:

- **LLM работает с текстом и кодом напрямую.** Агент может редактировать слайды, менять расположение блоков, править стили — всё через код.
- **Полный контроль.** Каждый элемент доступен для точечных изменений без перегенерации всего слайда.
- **Предсказуемость.** В отличие от генерации изображений, HTML-результат стабилен и воспроизводим.
- **Экспорт.** Слайды экспортируются в PNG через html2canvas, затем собираются в PDF.

## Как пользоваться

1. Подготовь текст презентации и положи в `wording/` (рекомендуется `.md`, см. «wording/»).
2. Положи визуальные референсы в `visual_refs/`.
3. Если есть дополнительные инструкции для агента, положи их в `skills/`.
4. Запусти LLM/coding agent в корне проекта.
5. Агент должен:
   - прочитать `AGENTS.md`;
   - проверить релевантные skills;
   - изучить `wording/`;
   - изучить `visual_refs/`;
   - создать или обновить `style_brief.md` на основе `visual_refs/`;
   - создать или обновить `image_manifest.json`;
   - создать или обновить `index.html`.
6. Сгенерируй изображения по `image_manifest.json` (см. «Генерация изображений»).
7. Положи готовые изображения в `imgs/`.
8. Попроси агента запустить локальный сервер (см. «Локальный сервер»).
9. Запусти экспорт PNG через кнопку в браузере.
10. Собери PNG в PDF (см. «Сборка PDF»).

## Локальный сервер

Не открывай `index.html` напрямую через `file://` — экспорт PNG через html2canvas не работает из-за ограничений браузера на кросс-доменные запросы и canvas.

Попроси агента запустить локальный сервер. Если в проекте есть `scripts/serve.py`:

```bash
python3 scripts/serve.py
```

Агент откроет ссылку вида `http://localhost:8000`. Открой её в браузере и экспортируй PNG.

## Генерация изображений

А генерирует изображения в LLM (DALL·E, Midjourney и т.д.) по промптом из `image_manifest.json`.

Пока изображения не сгенерированы, в `index.html` отображаются placeholder-блоки. Каждый placeholder содержит:

- описание изображения на русском;
- рекомендуемый размер;
- готовый промпт для генерации.

### Как работают промпты

Агент создаёт промпты по правилам из `skills/image-manifest-prompts.md`. Каждый промпт включает:

- что изображено;
- композицию;
- ссылку на стиль из `style_brief.md`;
- соотношение сторон;
- ограничения (например, запрет на текст внутри изображения).

Промпт всегда ссылается на стиль из `style_brief.md`, а не придумывает свой.

## Сборка PDF

После экспорта PNG-файлов слайдов нужно собрать их в PDF-презентацию.

1. Открой [ilovepdf.com/jpg_to_pdf](https://www.ilovepdf.com/jpg_to_pdf).
2. Загрузи все PNG-файлы слайдов.
3. Проверь порядок слайдов — нажми кнопку **A↓Z** для сортировки по имени.
4. Выбери размер страницы — **Fit (Same page size as image)**.
5. Выбери ориентацию — **Landscape**.
6. Нажми **Convert to PDF** и скачай готовый файл.

## Папки

### wording/

Готовый текст презентации.

Подходящие форматы:

- `.md` (рекомендуется);
- `.txt`;
- `.docx` (через конвертацию).

Рекомендуемый формат — `.md` со структурой:

```markdown
# Номер слайда. Заголовок / название

Текст слайда.

Описание желаемого вида и блоков.
```

Текст, цифры и факты считаются уже проверенными. Агенту запрещено менять их без явного разрешения.

### visual_refs/

Визуальные референсы:

- презентации;
- PDF;
- изображения;
- скриншоты;
- брендбуки;
- moodboards;
- понравившиеся примеры дизайна.

### imgs/

Готовые изображения, которые агент подставляет в презентацию, если они есть.

### skills/

Технические инструкции для агента. Описывают как работать с кодом, слайдами и экспортом, но не задают визуальный стиль.

Текущие файлы:

- **`html-structure.md`** — техническая структура `index.html`: visual slots, placeholder mechanics, требования к файлу (один файл, без сборки, без Node.js).
- **`png-export-safety.md`** — правила PNG-экспорта: запуск локального сервера, работа html2canvas, проверка экспортированных изображений.
- **`slide-layout-safety.md`** — layout слайдов: размеры 1280×720px, overflow, header/footer, визуальная проверка геометрии.
- **`image-manifest-prompts.md`** — правила создания `image_manifest.json`: структура записей, формат промптов, имена файлов, проверка перед сдачей.

Можно добавлять свои `.md`-файлы.
