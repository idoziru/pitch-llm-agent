# Презентации с помощью LLM-агента

Это проект для работы с LLM-агентом над презентацией.

На выходе вы получаете **`index.html` со слайдами**. Этот файл потом можно выгрузить в PNG и собрать в **финальный PDF**.

[Скачать архив проекта (.zip)](https://github.com/idoziru/pitch-llm-agent/archive/refs/heads/main.zip) — скачайте и распакуйте у себя на компьютере, чтобы начать.

## Какой агент подойдёт

Нужен LLM-агент, который умеет **работать с файлами проекта** (например, Cursor, Windsurf, Zed).

Внутри агента обязательно выберите **модель с поддержкой изображений** (например, GPT-4o, Claude Sonnet, Gemini 2.5 Pro). Без этого модель не сможет посмотреть ваши визуальные примеры из `visual_refs/`.

## Быстрый старт

1. Положите текст презентации в `wording/` (форматы: `.md` — рекомендуется, `.txt`, `.docx`).
2. Положите примеры стиля в `visual_refs/` (форматы: `.png`, `.jpg`, `.webp`, `.pdf`).
3. Откройте папку проекта в вашем агенте.
4. Напишите агенту: **«Сделай презентацию»**.
5. Дождитесь, пока агент сообщит, что презентация готова.
6. Напишите агенту: **«Запусти локальный сервер и дай ссылку для просмотра»**. По ссылке откройте презентацию и выгрузите слайды в PNG.
7. Соберите PNG в финальный PDF через ilovepdf (инструкция ниже).

## Как работать с картинками

Агент сам подготовит в презентации места под изображения и даст готовые промпты для генерации.

Дальше ваш процесс:

1. Сгенерируйте картинки по этим промптам в любом сервисе генерации изображений (например, ChatGPT/DALL·E, Midjourney, Gemini).
2. Сохраните их **с теми же именами файлов**, которые указал агент.
3. Положите файлы в папку `imgs/`.
4. Попросите агента обновить презентацию.

После этого картинки автоматически появятся в слайдах вместо заглушек.

## Как получить финальный PDF

1. Откройте [www.ilovepdf.com](https://www.ilovepdf.com/) и выберите инструмент **JPG to PDF**.
2. Загрузите PNG-файлы слайдов.
3. Нажмите сортировку **A↓Z**, чтобы порядок был правильный.
4. Выберите размер страницы: **Fit (Same page size as image)**.
5. Выберите ориентацию: **Landscape**.
6. Нажмите **Convert to PDF** и скачайте итоговый файл.

## Коротко про папки

- `wording/` — ваш текст для слайдов.
- `visual_refs/` — примеры стиля.
- `imgs/` — готовые картинки для вставки в презентацию.

Остальные файлы и папки в проекте — служебные, они нужны агенту. Трогать их не нужно.

## Что важно

- Агенту запрещено менять без вашего разрешения текст, цифры и факты из `wording/`.
- Размер слайдов фиксированный: **1280×720**. Это самый популярный формат слайдов.
- Если для запуска агенту понадобится что-то установить (например, Python) — попросите его подсказать, как это сделать.
- Вы можете добавлять свои правила для агента: создайте `.md`-файл в папке `skills/` и опишите внутри, что и как агент должен делать (например, «всегда оставляй последний слайд под контакты»). Агент будет учитывать эти файлы при работе.

## Рекомендуемый формат текста презентации

Лучше всего работает `.md`-файл со структурой по слайдам:

```markdown
# Слайд 1. Заголовок слайда

Основной текст слайда.

- Тезис 1
- Тезис 2

Комментарий по желаемому виду: «слева текст, справа картинка», «крупная цифра по центру» и т.п.

---

# Слайд 2. Следующий заголовок

...
```

Можно использовать `.txt` или `.docx`, но `.md` даёт самый предсказуемый результат, потому что агент чётко видит границы слайдов и структуру.

## Частые вопросы

- **Почему не открывается экспорт PNG?**
  Скорее всего, файл открыли напрямую, без сервера. Попросите агента запустить локальный сервер и откройте ссылку от агента.

- **Почему картинка не подставилась?**
  Обычно причина в имени файла. Проверьте, что имя в `imgs/` полностью совпадает с тем, которое указал агент.
