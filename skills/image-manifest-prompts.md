# image-manifest-prompts.md

## Назначение

Используй этот skill для создания и обновления `image_manifest.json`, placeholder-блоков и промптов для генерации иллюстраций.

Этот skill не задаёт визуальный стиль.

Стиль изображений должен браться из `style_brief.md`, который создан на основе `visual_refs/`.

---

## Общий принцип

Иллюстрации должны поддерживать смысл слайда, а не быть декоративной абстракцией.

Если слайд не требует изображения, не добавляй visual slot только ради заполнения места.

---

## Placeholder-first workflow

На первом этапе вместо иллюстраций вставляй placeholder-блоки.

Каждый placeholder должен быть на русском языке и содержать:

- подробное описание изображения;
- filename;
- aspect ratio;
- target size.

Описание должно быть достаточно конкретным, чтобы пользователь мог скопировать его в другую LLM и сгенерировать изображение.

Внешний вид placeholder бери из `style_brief.md`.

---

## image_manifest.json

`image_manifest.json` должен содержать все visual slots.

Каждая запись:

```json
{
  "slide": 2,
  "slot": "right_visual",
  "filename": "slide-02-customer-problem-map.png",
  "description_ru": "Схема пользовательской проблемы",
  "aspect_ratio": "4:3",
  "size": "560x420",
  "prompt_hint_ru": "Подробное описание изображения для генерации",
  "status": "placeholder"
}
```

Правила:

- не удаляй существующие записи без явного запроса пользователя;
- не пересоздавай весь файл, если можно обновить отдельные записи;
- `filename` должен точно совпадать с тем, что ожидает `index.html`;
- `description_ru` — короткое название изображения;
- `prompt_hint_ru` — подробное ТЗ для генерации;
- `prompt_hint_ru` должен ссылаться на стиль из `style_brief.md`, а не придумывать новый стиль;
- `aspect_ratio` должен соответствовать layout slot;
- `size` должен соответствовать реальному месту на слайде;
- `status` может быть `placeholder` или `found`.

---

## Имена файлов

Используй понятные имена:

```text
slide-XX-short-name.png
```

Примеры:

```text
slide-02-customer-problem-map.png
slide-04-market-structure.png
slide-07-product-flow.png
```

Не меняй filename, если пользователь уже мог сгенерировать изображение под старым именем.

---

## Требования к prompt_hint_ru

Промпт должен включать:

- что изображено;
- композицию;
- связь со стилем из `style_brief.md`;
- запрет на текст внутри изображения, если текст не нужен;
- соотношение сторон;
- важные ограничения.

Не задавай самостоятельный визуальный стиль, если он не взят из `style_brief.md`.

Плохой prompt:

```text
Красивая абстрактная иллюстрация про рынок.
```

Хороший prompt:

```text
Иллюстрация по стилю из style_brief.md: карта рынка из 4–5 сегментов, центральный сегмент выделен, остальные второстепенные; без текста внутри изображения; чистая композиция; соотношение 16:9.
```

---

## Проверка

Перед сдачей проверь:

- все visual slots есть в `image_manifest.json`;
- все filenames совпадают с `index.html`;
- prompt hints написаны на русском;
- prompts достаточно подробные для генерации;
- prompts не вводят стиль, которого нет в `style_brief.md`;
- старые записи не удалены без запроса;
- найденные изображения имеют `status: found`;
- отсутствующие изображения имеют `status: placeholder`.
