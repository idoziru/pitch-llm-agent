# research-sources.md

## Назначение

Используй этот skill для работы с **внешними источниками и поиском в интернете**. Он дополняет дисциплину источников из `pitch-wording.md` (source register, data status).

Этот skill не задаёт визуальный стиль и контент.

---

## Когда использовать поиск

Поиск в интернете запускается **только по явной просьбе пользователя**.

Агент может **предложить** поиск, когда это усилит питч, но не запускает его без согласия. Полезные поводы предложить:

- проверить цифру, бенчмарк или comps;
- проверить актуальность рыночных данных (размер рынка, CAGR, тренды);
- собрать контекст по аудитории, конкурентам или peers;
- проверить, не устарели ли данные (особенно старше 2–3 лет).

Формулируй предложение конкретно: «Могу проверить через поиск размер рынка за актуальный год — запустить?».

---

## Агент-агностичность

Не у каждого агента есть доступ к интернету.

- Если поиск доступен и пользователь его попросил — используй.
- Если поиск недоступен — сообщи об этом и пометь соответствующие данные `requires validation`. Не выдумывай данные взамен.

---

## Гарантии при использовании найденного

- Всегда фиксируй **источник и дату** в source register (см. `pitch-wording.md`).
- Найденное без проверяемой ссылки не выдавай за факт — помечай `requires validation`.
- Различай статус: `fact` (есть источник) / `management estimate` / `forecast` / `illustrative`.
- Для рыночных данных указывай год и период; для бенчмарков — методологию.
- Цифры старше 2–3 лет проверяй на актуальность.
- Не подменяй внутренние данные компании найденными в интернете без явной пометки, что это внешняя оценка.

---

## Конфиденциальность

- Не отправляй в поиск конфиденциальные данные клиента: внутренние финпоказатели, закрытые названия проектов, непубличные цифры.
- Если нужно что-то проверить, переформулируй запрос так, чтобы он не раскрывал конфиденциальные данные.
- Не превращай найденные данные в reusable benchmark, смешивая их с конфиденциальными (см. правила confidentiality в `pitch-wording.md`).
